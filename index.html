Getting started has never been easier.

Yet, it is flexible.

Traditional frameworks are rigid. After a certain point, as you scale, you end up spending more time fighting the framework than the time it saved you.

Stem is different because integrations are ejectable.

For example, if you eject stem-nodemailer then you have full control over the email integration. You can even eject stem-react and completely control how React is integrated with the rest of your stack. At that point, you can replace React with Vue.

As your project scales, and the need for custom integration arises, you eject the integrations you need to control.

We keep integrations simple with only a few lines of code. For example, $ npm run eject stem-nodemailer adds only 152 lines of code to your source code, and $ npm run eject stem-react only adds 384 lines.

Truly flexible

Stem is only a collection of integrations: if you eject all of your stem-* packages, then you completely remove Stem, and you are left with do-one-thing-do-it-well tools with full control over how they integrate with each other.

With Stem, you can get all the control you need. Without compromise.

No magic

Stem may seem magical — it isn't.

Integrations use a set of open standards enabling interoperability.

That's why, for example, you can replace stem-sqlite with stem-postgresql in a seamless way: your other integrations, such as your authentication integration, work just like before because they communicate with the database using an open standard that standarizes basic CRUD operations.

Simple

Everything we do is done with a meticulous care for simplicity.

Stem integrations are simple. Our recommended stacks are simple. Our do-one-thing-do-it-well tools are simple.

If you care about simplicity (and you should!) then you'll feel right at home.

Open

Integrations are based on open standards: there is no proprietary lock-in.

Actually, if you re-publish stem-* packages under another name, then you essentially clone the Stem project.

Not only that, but Stem is also decentral: stem-* package can be developed independently of each other.

Stem is, at its core, open and collaborative.
