Getting started has never been easier.

Yet, it is flexible.

Frameworks are rigid: after a certain point, as you scale, you end up spending more time fighting the framework than the time it saved you.

Stem is different: integrations are ejactable

For example, if you eject stem-nodemailer then you have full control over the emailing integration.

You can even eject stem-react to completely control how React is integrated with the rest of your stack.

As your project scales, and the need for custom integration arises, you eject the integrations you need to control.

The best part is that integrations are only few lines of code:

  • $ npm run eject stem-nodemailer adds only 152 lines of code to your source code.
  • $ npm run eject stem-react adds only 384 lines of code to your source code.

Truly flexible

Stem is only a collection of integrations: if you eject all of your stem-* packages, then you completely remove Stem. At that point, you are left with do-one-thing-do-it-well tools and have full control over how these tools integrate with each other.

With Stem, you can get all the control you need. Without compromise.

No magic

Stem may seem magical — it isn't.

Integrations use a set of open standards enabling interoperability.

That's why, for example, you can replace stem-sqlite with stem-postgresql in a seamless fashion: your other Stem integraions work just like before. (For example, Stem authentication integrations such as stem-nextauth communicate with the database using standarized CRUD database-agnostic operations.)

Simple

Everything we do is done with meticulous care for simplicity.

Stem integrations are simple. Our recommended stacks are simple. Our do-one-thing-do-it-well tools are simple.

If you care about simplicity (and you should!) then you'll feel right at home.

Open

Integrations are based on a open standard: there is no proprietary lock-in.

Stem is decentral: each stem-* package can be developed independently of each other.

Actually, if you re-publish stem-* packages under another name, then you essentially clone the Stem project.

Stem is, at its core, open and collaborative.
